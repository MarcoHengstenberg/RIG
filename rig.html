RIG

A CSS Framework by Codeship

What is RIG

Rig is a Basic CSS Framework made by the guys from codeship. It was created during the relaunch process while aiming for a modern UI with a well structured and clean CSS approach.

  • Thought through default values for many html elements
  • A flexible yet powerfull grid system
  • BEM Oriented CSS structure

Basics

When we started to create RIG it was based on the decission to keep the frontendfootprint as small as possible. The idea of taking a fully designed CSS Framework and afterwards override most of the values to differ from the original looks as much as possible was one thing we wanted to avoid. We think a CSS Framework should be as small and minimal as possible as it should only show you a direction and not force you into it.

The goal of RIG is to support you with clever and semantic classes wherever possible. Feel free to go through the sourcecode and alter things to your project needs. Also if you discover any bugs please feel free to open a GitHub Issue any time.

Responsive

This Framework aims for awesome responsive support as much as possible. It`s not perfect in any case but will grow to all the needs.

The Following Breakpoints and widths are defined by default and should help you provide readable content.

  • Contentwidth68rem

    This is the maximum Content width so it creates a good reading experience on very large Screens

  • Large> 60rem

    This is the first breakpoint. Everything bigger than 60rem counts as a large Screen

    Classes regarding large Screens contain lg

  • Medium45rem - 60rem

    This is the second breakpoint. Medium Screens usually should be readable between 45rem - 60rem

    Classes regarding medium Screens contain md

  • Small30rem - 45rem

    The third Breakpoint area aims to provide a good readability on Small Screens in 30rem - 45rem

    Classes regarding small Screens contain sm

  • Xtra Small< 30rem

    Xtra Small are all screens that are below 30rem

    Classes regarding very small Screens contain xs

Grid

The Gridsystem we have available througout the page should be very easy to use. It doesn't bring along a lot of styling so it only helps you keeping things in proportions you like. This system isn't build on columns as columns don't always make sense. Instead you can use percentages of 100% space

Available Gridspaces

You can use grid columns width values 10%, 15%, 20%, 25%, 30%, 33%, 40%, 50%, 60%, 66%, 70%, 75%, 80%, 85%, 90% and 100%

You can also use size--offset00 or size--tail00 to prepend or append column spaces.

To create a column just add the class .col to the element and append the according column sizes.

The following example shows the same grid col size on every screen width. If you don't set a width for certain sizes it will be 100% by default

Special Grid Classes

There are some special classes that will help you work with grids more effectively.

  • .grid--cling

    This will remove the padding on both sides of the grid giving you full when needed. For Example Images or colored containers.

  • .col--cling

    This removes the padding on a column.
